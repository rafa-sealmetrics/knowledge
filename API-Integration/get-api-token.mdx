# Getting Your Sealmetrics API Token

Learn how to generate, manage, and secure your Sealmetrics API tokens for programmatic access to your analytics data.

## What is an API Token?

An API token is a secure credential that allows applications to access your Sealmetrics data programmatically. It acts as a digital key that authenticates your requests to our API.

### Token Characteristics
- **Unique Identifier**: Each token is unique to your account
- **Secure Access**: Provides authenticated access to your data
- **Configurable Permissions**: Can be limited to specific data types
- **Revocable**: Can be disabled or regenerated at any time

## Step-by-Step Token Generation

### Step 1: Access Your Dashboard
1. Log into your Sealmetrics account
2. Navigate to the main dashboard
3. Ensure you have admin or API access permissions

### Step 2: Navigate to API Settings
1. Click on your profile icon (top right)
2. Select "Account Settings" from dropdown
3. Click on "API & Integrations" tab
4. Look for "API Tokens" section

### Step 3: Create New Token
1. Click "Generate New Token" button
2. Provide a descriptive name for the token
   - Example: "Marketing Dashboard Integration"
   - Example: "Monthly Reporting Script"
   - Example: "CRM Data Sync"

### Step 4: Configure Permissions
Select the appropriate permissions for your use case:

#### Read Permissions
- **Analytics Data**: Access to traffic and conversion data
- **Account Info**: Basic account information
- **Reports**: Access to generated reports

#### Write Permissions
- **Goals**: Create and modify conversion goals
- **Settings**: Update account settings
- **Users**: Manage team member access

### Step 5: Generate and Copy Token
1. Click "Generate Token"
2. **Important**: Copy the token immediately
3. Store it securely (you won't see it again)
4. Click "Done" to complete the process

## Token Management Best Practices

### Naming Conventions
Use descriptive names that indicate:
- **Purpose**: What the token is used for
- **Application**: Which app or script uses it
- **Environment**: Production, staging, or development
- **Team**: Which team member or department owns it

#### Good Examples
- `production-dashboard-readonly`
- `marketing-reports-monthly`
- `crm-integration-fullaccess`
- `dev-testing-limited`

#### Poor Examples
- `token1`
- `mytoken`
- `test`
- `backup`

### Security Guidelines

#### Storage Security
- **Environment Variables**: Store in environment variables
- **Secret Management**: Use dedicated secret management tools
- **No Version Control**: Never commit tokens to Git repositories
- **Encrypted Storage**: Encrypt tokens at rest

#### Access Control
- **Principle of Least Privilege**: Grant minimal necessary permissions
- **Regular Audits**: Review token usage regularly
- **Team Access**: Limit who can generate tokens
- **Documentation**: Document token purposes and owners

#### Rotation Policy
- **Regular Rotation**: Rotate tokens every 90 days
- **Immediate Rotation**: Rotate if potentially compromised
- **Planned Rotation**: Schedule rotation during maintenance windows
- **Emergency Procedures**: Have emergency rotation procedures

## Using Your Token

### Authentication Header Format
```
Authorization: Bearer YOUR_API_TOKEN
```

### Example API Calls

#### cURL Example
```bash
curl -H "Authorization: Bearer sm_1234567890abcdef" \
     -H "Content-Type: application/json" \
     https://api.sealmetrics.com/v1/accounts
```

#### JavaScript Example
```javascript
const token = process.env.SEALMETRICS_API_TOKEN;

const response = await fetch('https://api.sealmetrics.com/v1/analytics', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

#### Python Example
```python
import os
import requests

token = os.environ.get('SEALMETRICS_API_TOKEN')
headers = {
    'Authorization': f'Bearer {token}',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.sealmetrics.com/v1/analytics', headers=headers)
```

## Troubleshooting Common Issues

### Token Not Working

#### Check Token Format
- Ensure "Bearer " prefix in Authorization header
- Verify no extra spaces or characters
- Confirm token is complete (not truncated)

#### Verify Permissions
- Check if token has required permissions
- Confirm account access is still valid
- Verify token hasn't been revoked

#### Test with Simple Call
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
     https://api.sealmetrics.com/v1/accounts
```

### Rate Limiting Issues
- **Symptom**: 429 Too Many Requests errors
- **Solution**: Implement exponential backoff
- **Prevention**: Respect rate limits (100 requests/minute)

### Permission Errors
- **Symptom**: 403 Forbidden responses
- **Solution**: Check token permissions
- **Fix**: Generate new token with correct permissions

## Security Incident Response

### If Token is Compromised
1. **Immediate Action**: Revoke the compromised token
2. **Generate New**: Create replacement token
3. **Update Applications**: Deploy new token to applications
4. **Monitor**: Watch for unusual API activity
5. **Document**: Record incident for future reference

### Revocation Process
1. Go to API Settings in your dashboard
2. Find the compromised token in the list
3. Click "Revoke" or "Delete"
4. Confirm the action
5. Update all applications using that token
