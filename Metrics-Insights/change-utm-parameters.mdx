# How to change UTM parameters

This guide helps customers using custom parameters or Adobe Analytics (rather than Google Analytics) to configure SealMetrics with their preferred parameter structure for marketing campaign tracking.

## When You Need Custom UTM Parameters

### Alternative Analytics Platforms

- **Adobe Analytics**: Uses different parameter conventions
- **Custom tracking systems**: Internal parameter structures
- **Enterprise solutions**: Company-specific tracking requirements
- **Legacy systems**: Existing parameter conventions to maintain
- **Multi-platform tracking**: Consistent parameters across platforms

### Business Requirements

- **Naming conventions**: Company-specific parameter naming
- **Campaign structure**: Unique campaign organization needs
- **Reporting consistency**: Maintain consistency across tools
- **Team workflows**: Match existing team processes
- **Historical data**: Preserve historical parameter usage

## Standard vs Custom UTM Parameters

### Standard Google Analytics UTMs

```
utm_source=google
utm_medium=cpc
utm_campaign=summer_sale
utm_content=banner_ad
utm_term=running_shoes
```

### Adobe Analytics Examples

```
cid=summer_sale_google_cpc
s_kwcid=AL!1234!3!567890!e!!g!!running%20shoes
ef_id=abc123def456:G:s
s_cmp=summer_sale
```

### Custom Enterprise Examples

```
src=google
med=paid_search
camp=summer_sale_2024
cont=banner_300x250
kw=running_shoes
```

## Common Custom Parameter Structures

### Adobe Analytics Integration

**Adobe standard parameters:**

- `s_cmp` → Campaign name
- `s_kwcid` → Keyword tracking
- `cid` → Campaign ID
- `ef_id` → External ID tracking

**Configuration mapping:**

```
Standard UTM → Adobe Parameter
utm_source → cid (source portion)
utm_medium → s_cmp (medium portion)
utm_campaign → s_cmp (campaign portion)
utm_content → s_kwcid (content tracking)
utm_term → s_kwcid (keyword tracking)
```

### Enterprise Custom Parameters

**Shortened parameter names:**

```
src → Source
med → Medium  
cmp → Campaign
cnt → Content
trm → Term
```

**Company-specific conventions:**

```
channel → Traffic source
campaign_id → Campaign identifier
creative_id → Creative variation
placement → Ad placement
audience → Target audience
```

## Advanced Configuration

### Dynamic Parameter Generation

```
// Generate parameters based on context
function generateCustomParams(campaign, source, content) {
    const params = new URLSearchParams();
    params.append('cmp', campaign);
    params.append('src', source);
    params.append('cnt', content);
    params.append('ts', Date.now());
    return params.toString();
}
```
