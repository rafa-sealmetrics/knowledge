# How to add a new account

Adding new accounts in SealMetrics enables you to track multiple websites, business units, or client projects separately, all while managing everything from a single dashboard.

## When to Add New Accounts

### Multiple Websites

- **Different domains**: Separate tracking for different websites
- **Brand separation**: Keep different brands' data separate
- **Client management**: Agency use case for multiple clients
- **Business units**: Different departments or divisions

### Organizational Benefits

- **Data isolation**: Keep different projects' data separate
- **Access control**: Grant team members access to specific accounts
- **Reporting clarity**: Generate account-specific reports
- **Billing transparency**: Track usage per account

## Step-by-Step Process

### Step 1: Access Account Management

Navigate to the accounts section in your SealMetrics dashboard.

### Step 2: Create New Account

1. Click **"\+ New Account"** or **"Add Account"**
2. Fill in the required information:
   - **Account name**: Descriptive name for identification
   - **Domain**: Primary domain for this account
   - **Description**: Optional description for account purpose
   - **Time zone**: Set appropriate time zone for the account

### Step 3: Configure Account Settings

- **Tracking setup**: Get tracking code for the new account
- **Goal configuration**: Set up conversion goals
- **Team access**: Assign users to the new account
- **Notification settings**: Configure email notifications

### Step 4: Implement Tracking

- **Install tracking code**: Add SealMetrics code to your website
- **Test implementation**: Verify tracking is working correctly
- **Configure domains**: Add all relevant domains to the account
- **Set up goals**: Configure conversion tracking

## Account Configuration Options

### Basic Settings

- **Account name**: Display name for internal use
- **Primary domain**: Main website domain
- **Account slug**: URL-friendly identifier
- **Description**: Account purpose and details
- **Time zone**: Local time zone for reporting

### User Management

- **Admin users**: Users with full account access
- **Editor users**: Users who can modify settings
- **Account-specific access**: Limit users to specific accounts

## Best Practices

### Naming Conventions

- **Descriptive names**: Use clear, descriptive account names
- **Consistent format**: Maintain consistent naming across accounts
- **Client identification**: Include client name for agency use
- **Project codes**: Use internal project codes if applicable

### Organization Structure

- **Logical grouping**: Group related accounts together
- **Access planning**: Plan user access before creating accounts
- **Documentation**: Maintain records of account purposes
- **Regular review**: Audit accounts periodically

## Managing Multiple Accounts

### Dashboard Navigation

- **Account switcher**: Easy switching between accounts
- **Unified view**: See all accounts from main dashboard
- **Quick access**: Bookmark frequently used accounts
- **Search functionality**: Find accounts quickly

### Billing and Usage

- **Usage tracking**: Monitor analytics usage per account
- **Cost allocation**: Understand costs per account
- **Plan management**: Upgrade plans as accounts grow
- **Resource optimization**: Optimize usage across accounts

## Troubleshooting

### Common Issues

- **Tracking conflicts**: Multiple tracking codes on same site
- **Access problems**: Users can't access specific accounts
- **Data inconsistencies**: Different data between accounts
- **Configuration errors**: Incorrect account settings

### Solutions

- **Code review**: Verify tracking implementation
- **User management**: Check user permissions and access
- **Data validation**: Compare data sources for consistency
- **Support contact**: Contact SealMetrics support for complex issues
